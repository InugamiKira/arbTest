## 三、数据采集流程

### 每日数据更新流程

```
1. 启动数据更新器 (jsl001_data_updater.py)
       │
       ▼
2. 读取基金配置 (fund_list.csv)
       │
       ▼
3. 获取基金实时行情 (东财API)
       │
       ▼
4. 获取关联指数行情 (东财API)
       │
       ▼
5. 获取深交所场内份额 (深交所API)
       │
       ▼
6. 计算静态估值
       │
       ▼
7. 保存到数据库 (jsl_monitor.db)
       │
       ▼
8. 更新监控页面
```

### 静态估值计算流程

```
静态估值 = 前一日净值 × (1 + 仓位比例 × 指数涨跌幅)

其中：
- 前一日净值：从基金净值数据获取
- 仓位比例：从基金配置读取（默认95%）
- 指数涨跌幅：从东财API获取当日涨跌幅
```

## 四、关键设计原则

### 1. 数据格式统一
- 指数代码统一使用东财格式（无前缀）
- 价格数据统一处理（东财数据需÷100）
- 日期格式统一为YYYY-MM-DD

### 2. 防刷保护
- 每日只同步一次数据
- 使用数据库记录同步状态
- 请求间隔至少20秒

### 3. 错误处理
- 网络超时重试机制
- 数据库连接异常处理
- 数据缺失降级处理
